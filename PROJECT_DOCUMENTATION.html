Executive Summary

Database Models

User Model

{ _id: ObjectId, name: String, email: String (unique), role: String ('Admin' | 'Driver' | 'Customer'), authMethod: String (default: 'Passkey'), recoveryEmail: String, createdAt: Date }

Booking Model

{ _id: ObjectId, userId: ObjectId (ref: User), customerName: String, origin: String, destination: String, status: String ('Pending' | 'Completed'), createdAt: Date, updatedAt: Date }

Vehicle Model

{ _id: ObjectId, vehicleNumber: String (unique), vehicleType: String ('Truck' | 'Van' | 'Bike'), driverName: String, location: String, status: String ('Available' | 'In Transit' | 'Maintenance'), updatedAt: Date }

AuditLog Model

{ _id: ObjectId, userEmail: String, action: String ('REGISTER' | 'LOGIN' | 'BOOKING_CREATE' | ...), details: String, ipAddress: String, createdAt: Date }

Security Features

  • Passwordless Authentication: Email-based passkey simulation for educational purposes
  • JWT Tokens: Cryptographically signed, 8-hour expiry
  • HTTP-only Cookies: Prevents JavaScript XSS attacks
  • Role-Based Middleware: Enforces permissions at API level
  • Audit Logging: Every action logged with user, timestamp, IP
  • Helmet: Security headers protection (XSS, clickjacking, etc.)
  • CORS: Credential-enabled with origin verification
  • Rate Limiting: 200 requests per 15 minutes per IP

Authentication Flow

Registration Flow

  1. User fills form: Name, Email, Role, Recovery Email
  2. Frontend POSTs to /api/auth/register
  3. Backend validates unique email
  4. Backend creates User document in MongoDB
  5. Audit log entry created: action: 'REGISTER'
  6. Response: User object with ID, name, email, role
  7. User can now proceed to login

Login Flow

  1. User enters email address
  2. Frontend POSTs to /api/auth/login
  3. Backend finds user by email
  4. Backend generates JWT signed with JWT_SECRET
  5. JWT set as auth_token HTTP-only cookie
  6. Frontend stores token in localStorage
  7. User redirected to Dashboard

Protected Route Access

  • Auth middleware intercepts API requests
  • Reads JWT from cookie or Authorization: Bearer <token> header
  • Verifies signature and expiry using JWT_SECRET
  • Attaches decoded user data to req.user
  • If invalid/expired: returns 401 Unauthorized

Role-Based Access Control

Role Permissions
Admin Create/edit/delete vehicles, view all logs, access dashboard stats
Driver View assigned shipments, update shipment status
Customer Create and view own bookings only

How to Run the Project

Prerequisites

  • Node.js v18 or higher
  • npm (comes with Node.js)
  • MongoDB running locally on 127.0.0.1:27017

Installation Steps

1. Install Dependencies

npm install

2. Configure Environment Variables

Edit the .env file:

PORT=4000
MONGO_URI=mongodb://127.0.0.1:27017/securetms_tms
JWT_SECRET=myverysecretkey123
SESSION_EXPIRES_IN=8h

3. Start the Server

npm start

Or for development with auto-reload:

npm run dev

4. Open in Browser

http://127.0.0.1:4000

(If port is busy, the app will automatically use 4001, 4002, etc.)

Using the Application

  1. Register: Click "Create Account" and fill the form
  2. Login: Sign in with your registered email
  3. Dashboard: View and manage bookings, shipments, fleet
  4. Add Data: Use action buttons to create new records
  5. Logout: Click Logout in dashboard header
